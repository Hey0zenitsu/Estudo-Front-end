Parâmetros em JavaScript

Os parâmetros em JavaScript são usados para passar valores para funções. Eles permitem que uma função seja mais flexível, pois os valores podem variar a cada chamada. Vamos explorar os principais aspectos dos parâmetros:

1. Definição de Parâmetros

Os parâmetros são definidos dentro dos parênteses de uma função. Eles são variáveis locais que recebem os valores (argumentos) fornecidos na chamada da função.

Exemplo:

Exemplo: 
function saudacao(nome) {
                console.log(`Olá, ${nome}!`);
            }
            saudacao("Bruno"); // Saída: Olá, Bruno!
1

2. Parâmetros Padrão

Desde o ES6, é possível definir valores padrão para os parâmetros. Caso nenhum argumento seja fornecido, o valor padrão será utilizado.

Exemplo:

Exemplo 
function saudacao(nome = "Visitante") {
                console.log(`Olá, ${nome}!`);
            }
            saudacao(); // Saída: Olá, Visitante!
1

3. Parâmetros Rest (...)

O operador rest (...) permite agrupar múltiplos argumentos em um único array. Ele é útil quando você não sabe quantos argumentos serão passados.

Exemplo:

Exemplo 
function soma(...numeros) {
                return numeros.reduce((total, num) => total + num, 0);
            }
            console.log(soma(1, 2, 3, 4)); // Saída: 10
1

4. Parâmetros vs Argumentos

  • Parâmetros:: são as variáveis definidas na função.
  • Argumentos:: são os valores reais passados quando a função é chamada.

Exemplo:

Exemplo 
function multiplicar(a, b) {
                return a * b; // 'a' e 'b' são parâmetros
            }
            console.log(multiplicar(5, 4)); // 5 e 4 são argumentos
1

5. Ordem dos Argumentos

A ordem dos argumentos importa. Eles serão atribuídos aos parâmetros na mesma sequência.

Exemplo:

Exemplo 
function calcularArea(base, altura) {
                return base * altura;
            }
            console.log(calcularArea(5, 10)); // Saída: 50
1

6. Parâmetros Nomeados (Simulação)

JavaScript não possui suporte nativo para parâmetros nomeados como outras linguagens, mas isso pode ser simulado usando objetos.

Exemplo:

Exemplo 
function criarUsuario({ nome, idade, email }) {
                console.log(`Nome: ${nome}, Idade: ${idade}, Email: ${email}`);
            }
            criarUsuario({ nome: "Bruno", idade: 18, email: "bruno@email.com" });
1

7. Passagem de Valores

  • Por valor:: Para tipos primitivos (números strings etc.) os valores são copiados.
  • Por referência:: Para objetos e arrays a referência é passada e alterações no parâmetro afetam o valor original.

Exemplo:

Exemplo 
function alterarObjeto(obj) {
                obj.nome = "Alterado";
            }
            let usuario = { nome: "Original" };
            alterarObjeto(usuario);
            console.log(usuario.nome); // Saída: Alterado
1

8. Parâmetros Opcionais

Em JavaScript, parâmetros podem ser opcionais. Basta verificar se eles foram fornecidos.

Exemplo:

Exemplo 
function cumprimentar(nome) {
                if (nome) {
                    console.log(`Olá, ${nome}!`);
                } else {
                    console.log("Olá, visitante!");
                }
            }
            cumprimentar(()); // Saída: Olá, visitante!
1

Os parâmetros são essenciais para criar funções dinâmicas e reutilizáveis em JavaScript. Eles permitem que você passe informações para as funções, adaptando-as a diferentes situações. Aqui estão os principais pontos:

  • Definição: Os parâmetros são variáveis usadas dentro das funções.
  • Padrão: Valores podem ser definidos caso nenhum argumento seja passado.
  • Operador Rest: Agrupe múltiplos argumentos em um array com ....
  • Flexibilidade: Parâmetros opcionais e objetos permitem maior controle.
  • Por Valor e Por Referência: Tipos primitivos são copiados enquanto objetos e arrays compartilham a referência.

Ao dominar os parâmetros, você consegue criar funções mais adaptáveis, garantindo que elas se comportem bem em diversos cenários. Continue explorando, praticando e experimentando!
